## GIT FLOW & COMMIT RULES (CRITICAL)

**TRIGGER:** Apply these rules EVERY TIME you create a branch or generate a commit message.

1. **NO CYRILLIC:** English only in all branch names and commit messages.
2. **BRANCH NAMING:** Strictly use these prefixes:
   `feature/*`, `refactor/*`, `fix/*`, `chore/*`, `build/*`, `style/*`, `docs/*`, `release/*`
3. **CONVENTIONAL COMMITS:** You must format commits as `<type>(<optional scope>): <description>`.
    * Allowed types: `feat`, `fix`, `chore`, `refactor`, `test`, `docs`, `style`, `ci`, `perf`,
      `build`, `revert`.
4. **LOWERCASE ENFORCEMENT (FATAL ERROR IF IGNORED):**
   The description part MUST start with a lowercase letter. Do not capitalize the first word!
    * BAD: `feat(auth): Add login`
    * BAD: `ci: Refine git linting`
    * GOOD: `feat(auth): add login`
    * GOOD: `ci: refine git linting`
